Create a 10-slide hackathon pitch deck for "Kirana Commerce" — a multilingual (English + Telugu) web platform that digitizes the WhatsApp/manual ordering and delivery workflow used by Indian neighborhood kirana (grocery) stores, without changing how the business actually operates. Use a clean, minimalistic, professional style — white/grey/black palette, no clutter, tech-startup aesthetic. Structure:

1. Title slide: "Kirana Commerce" — Digitizing the Neighborhood Kirana Store. Subtitle: a multilingual ordering & delivery management platform. Theme: Business Digitization.

2. Problem: Kirana stores run their entire ordering process over WhatsApp/phone calls — manual notes, manual billing, no order history, no delivery tracking, frequent disputes and errors, and no support for the bilingual (English/Telugu) way customers actually communicate. Existing quick-commerce apps (Swiggy Instamart, Blinkit) don't fit a single local store's real workflow.

3. Solution: Kirana Commerce replicates the exact WhatsApp-simple experience customers already expect, but structured and digital — phone-based login with zero friction (no OTP, no password), full English/Telugu support including auto-translation, live map-based delivery addresses, and a real admin system replacing manual bookkeeping.

4. Who it's for — four roles: Customer (browse, order, track), Store Admin (products, orders, delivery, analytics), Delivery Partner (assigned deliveries, navigation, payment collection), Superadmin (multi-store platform oversight).

5. Key features — customer side: multilingual product search, cart with variants/quantities, phone-based checkout (no OTP), live GPS/map address picker, Cash/UPI/Bank Transfer payment, order history.

6. Key features — admin side: product & category management with one-click English-to-Telugu auto-translation, image upload pipeline, live analytics dashboard, delivery assignment on an interactive map, delivery partner management.

7. Key features — delivery partner side: assigned-deliveries dashboard, one-tap Google Maps navigation, in-transit/delivered status updates, cash-on-delivery confirmation.

8. Tech stack: Next.js 15 (React/TypeScript), PostgreSQL (Neon) for data, Redis (Upstash) for rate limiting and session caching, Bunny CDN + Sharp for image processing, Leaflet/OpenStreetMap + Ola Maps for live location and address autocomplete, Sentry for error monitoring — a production-grade architecture, not a mocked demo. Fully responsive design, deployed live on Vercel.

9. Live demo: show the deployed link (https://kirana-app-eight.vercel.app) with a screenshot collage — customer storefront, checkout with map picker, admin dashboard, admin product form with the Telugu translate button, delivery partner view.

10. Roadmap & closing: Android app (Capacitor wrapper), advanced multilingual search (Meilisearch), GitHub CI/CD pipeline, richer analytics. Closing line: "Kirana Commerce — bringing India's neighborhood stores online, on their own terms."

Keep text per slide concise (bullet points, not paragraphs). Use icons for the feature slides. Suggest a simple grayscale/monochrome icon set to match the app's own minimalistic design language.
